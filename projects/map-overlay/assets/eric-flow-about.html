Eric Flow

Digital Stream Table

A real-time particle simulation that traces how water flows downhill across actual terrain. Pan to any landscape on Earth, and watch drainage networks emerge from nothing but elevation data and gravity.

Eric Flow is a browser-based counterpart to the physical stream tables used in geomorphology and hydrology courses — sand-and-water models where students pour water over sculpted terrain and observe where channels form. Here, the sand is replaced by 30-meter Terrarium elevation tiles from AWS, and the water is replaced by thousands of particles following gradient descent across the DEM.

How It Works

Elevation tiles are fetched from the Mapzen Terrarium dataset at the current map extent. Each pixel encodes elevation in meters via RGB channels. Particles spawn along the top and left edges of the viewport — simulating water entering from high-elevation boundaries — and each frame, every particle samples elevation in four cardinal directions and accelerates toward the steepest descent. A damping factor simulates friction. The result is emergent drainage: linear channel patterns appear without any explicit channel geometry in the code.

Three particle size classes model sediment grain-size sorting in fluvial systems. Small particles (blue, 2px) have low inertia and follow every fine gradient. Medium particles (orange, 4px) carry moderate momentum. Large particles (pink, 6px) resist turning and travel straighter paths, analogous to coarse sediment transport.

The heat map accumulates particle visits over time, functioning as a visual analog to flow accumulation rasters computed from DEMs in GIS. Colors ramp from blue (low) through green and yellow to red (peak flow). Adjusting the decay slider controls whether you see recent flow paths or cumulative drainage patterns.

Layer Tree

The interface is organized as a layer tree — a pattern borrowed from GIS and adapted here for interactive simulation control.

📂 Eric Flow
⚙ Particles — Flow Rate, Density, Damping, Particle Size
🌡️ Heat Map — toggleable, with Decay and Clear
📂 Elevation
⚙ Terrarium Elevation (30m) — Opacity, Color Ramp

All control state persists to localStorage and restores on reload.

View Modes

2D Map, 3D Terrain (with adjustable exaggeration), and Globe projection. Six basemaps from satellite imagery to topographic maps. In 3D Terrain mode, the elevation exaggeration reveals the relationship between particle flow patterns and the underlying landform.

Concepts Demonstrated

Gradient descent
The same principle behind D8 and D-infinity flow routing algorithms. Each particle follows the local steepest-descent vector, producing realistic drainage without global path planning.
Flow accumulation
The heat map reveals where channels form and merge, replicating the core computation of hydrological GIS without ever running a formal flow accumulation algorithm.
Channel emergence
Linear drainage networks appear from simple local rules. No channels are drawn; they self-organize from particle trajectories.
Scale and resolution
Zooming in loads higher-resolution elevation tiles, revealing finer hydrological structure. Zooming out shows regional drainage basins.

How This Was Made

Ambient Authority Editing

This simulation was authored collaboratively between a human and an AI (Claude), with the AI reading, modifying, and writing files directly on a live Acequia WebDAV server — using nothing but the user’s existing browser session for authentication.

The pattern is described in detail at: ambient-authority-editing.md

The core idea: the user is already authenticated to the Acequia server via a browser cookie. The AI, operating within that browser context, inherits that ambient authority. It can GET a file, modify its contents, and PUT it back — standard WebDAV verbs on the actual resource URL. No API key is exchanged. No OAuth flow is triggered. No credentials are visible to the AI. The URL is the file, and HTTP verbs are the edit operations.

The collaboration unfolded as a series of conversational edits:

  1. The original README told readers to “paste script tags into the browser console” — which doesn’t work. The AI explained the difference, then edited and saved the fix via PUT.
  2. Play/pause button added, default map center moved to Ruidoso, NM. One fetch, one PUT.
  3. Before destructive edits, the AI backed up the file using a WebDAV COPY — server-side copy, zero bytes transferred.
  4. The Charlestown layer tree pattern was studied in a second browser tab, then rebuilt into eric-flow’s control panel with the same visual language.
  5. Terrarium Elevation visualization added — reusing existing DEM data as a color-ramped canvas overlay.
  6. Layer tree restructured into two top-level folders, control state wired to localStorage for persistence.

Every edit happened live on the server. The AI never saw a password. The entire authoring workflow used three HTTP verbs — GETPUTCOPY — that WebDAV has provided since 1999.

Why This Matters for Design

Technology is a landscape. The patterns we choose for how tools, data, and people connect shape what communities can build and observe. The Acequia platform treats files as URLs and authentication as ambient context — which means the threshold for collaborative making drops to nearly zero. An AI assistant, a student, a researcher, and a community modeler can all work on the same artifact, at the same URL, using the same verbs, without configuring anything.

For landscape architecture students: this is what it looks like when the infrastructure disappears and the conversation becomes the design tool. The stream table is both the subject (a hydrological model) and the medium (a web document, edited in place, on a shared server, by a human-AI pair). The observation platform and the thing being observed are the same artifact.
